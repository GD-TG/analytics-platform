# Исправление ошибки "Failed to fetch"

## Что было исправлено:

1. ✅ Создан файл `config/cors.php` с правильными настройками CORS
2. ✅ Улучшена обработка ошибок в HTTP клиенте
3. ✅ Добавлена лучшая обработка ошибок в контроллерах авторизации
4. ✅ Добавлены понятные сообщения об ошибках

## Проверка:

1. **Убедитесь, что backend запущен:**
   ```bash
   php artisan serve
   ```
   Должен быть доступен на http://localhost:8000

2. **Убедитесь, что frontend запущен:**
   ```bash
   cd frontend
   npm run dev
   ```
   Должен быть доступен на http://localhost:5173

3. **Проверьте настройки CORS:**
   - Файл `config/cors.php` должен содержать `http://localhost:5173` в `allowed_origins`

4. **Очистите кеш конфигурации:**
   ```bash
   php artisan config:clear
   php artisan cache:clear
   ```

## Если ошибка все еще возникает:

1. **Проверьте консоль браузера (F12)** - там будут детали ошибки
2. **Проверьте Network tab** - посмотрите, какой запрос не проходит
3. **Убедитесь, что оба сервера запущены** и доступны
4. **Проверьте, что база данных настроена** и миграции выполнены

## Типичные проблемы:

- **Backend не запущен** → Запустите `php artisan serve`
- **Неправильный URL API** → Проверьте `VITE_API_URL` в `.env` frontend или используйте значение по умолчанию `http://localhost:8000`
- **CORS блокирует запросы** → Убедитесь, что `config/cors.php` настроен правильно
- **База данных не настроена** → Выполните миграции: `php artisan migrate`
